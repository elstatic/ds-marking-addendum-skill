# Памятка для пользователя: запуск с нуля

Эта инструкция для пользователя, который не работает с терминалом.

## 1) Установить Google Drive

Скачайте и установите Google Drive for desktop:
[https://www.google.com/drive/download/](https://www.google.com/drive/download/)

Прямые ссылки на установщики:
- Windows: [GoogleDriveSetup.exe](https://dl.google.com/drive-file-stream/GoogleDriveSetup.exe)
- macOS: [GoogleDrive.dmg](https://dl.google.com/drive-file-stream/GoogleDrive.dmg)

Войдите в рабочий Google-аккаунт и дождитесь синхронизации папки с договорами.

## 2) Добавить папку в Codex как проект

Откройте Codex и добавьте проектом папку:
`.../Договоры/Договоры (по клиентам)`

## 3) Попросить агента установить навык

Напишите агенту:

```text
Установи мне навык из репозитория:
https://github.com/elstatic/ds-marking-addendum-skill
```

После установки перезапустите Codex.

## 4) Передать шаблон и задачу на подготовку ДС

Отправьте агенту шаблон ДС (файл `.docx` или ссылку на Google Doc), затем команду:

```text
Подготовь ДС на администрирование маркировки рекламных материалов по этому шаблону
для клиентов:
1) <клиент 1>
2) <клиент 2>
Номер ДС: <номер или правило нумерации>
Дата подписания: <ДД.ММ.ГГГГ>
Сохрани готовые файлы в папках соответствующих клиентов и пришли пути к файлам.
```

## Что должно получиться

Агент:
- найдёт договоры клиентов в папках;
- подставит реквизиты и подписанта;
- сформирует готовые `.docx` ДС;
- сохранит их в папках соответствующих клиентов;
- пришлёт вам пути к созданным файлам.
